# Security Policy

## Supported Versions

This repository is an experimental Coding Academy static React project.

| Version / Branch | Supported | Notes |
| --- | --- | --- |
| `main` | Yes | Latest public source snapshot for review and learning |
| Older commits and branches | No | Historical snapshots are not actively maintained |

## Reporting a Vulnerability

Please do not disclose security issues publicly in GitHub issues, pull requests, or discussions.

If GitHub's private vulnerability reporting flow is available for this repository, use the **Report a vulnerability** option in the repository security interface.

If private vulnerability reporting is not available, open a minimal GitHub issue that does **not** include exploit details, secrets, payloads, or reproduction steps that would put users at risk. Use that issue only to request a private follow-up path before sharing sensitive details.

## What to Report

Report issues such as:

- exposed secrets or credentials
- cross-site scripting or HTML injection risks
- insecure data handling in client-side storage
- unsafe third-party library behavior that affects this repository

## Response Expectations

Because this is an archived learning project, response times are best effort and fixes are not guaranteed. Valid reports will still be reviewed and triaged as time allows.
